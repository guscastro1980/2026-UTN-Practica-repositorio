## Git Flow

Con gitflow aprenderemos a manejar un repositorio colaborativo.

Nunca trabajamos sobre la rama funcional main, ya que estos cambios pueden tener errores y contener fallos. Para evitarlo creamos una subrrama llamada development o dev

Para crear la rama de desarrollo usamos el comando: git checkout -b <nombre de la nueva rama>
nos crea y nos mueve a la rama de desarrollo.
